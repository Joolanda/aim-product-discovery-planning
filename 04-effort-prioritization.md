# 04 – Effort-Informed Prioritization

## 💡 Why this exercise matters
Not everything fits in three weeks. This matrix keeps the team honest about what to deliver now vs later.

## 🎯 Expected Outcomes
- Prioritized feature set within realistic capacity  
- Balanced learning and user value  

---

### 🔧 Prompt – Prioritize by Value × Effort
List all features and their estimated effort, then classify:

| Priority | Definition | Approx Effort |
|-----------|-------------|----------------|
| **MUST HAVE** | Core to MVP (user value proof) | 8–10 days |
| **SHOULD HAVE** | Important but not critical | 3–5 days |
| **COULD HAVE** | Nice-to-have if time left | 1–2 days |
| **WON’T HAVE** | Explicitly deferred | – |

✅ Ensure total ≤ 15 team days + 20 % buffer.  
Then translate prioritized items into the structure:  
**Outcome Goal → Epic → Deliverables → Tasks**

---

### 🧩 Example – *DataPath AI* MVP

| Feature | Estimated Effort | Priority | Reason |
|----------|------------------|-----------|---------|
| Narrative → Skills Extractor (text → skills graph) | 7 days | ✅ **MUST HAVE** | Core value; proves AI can extract skills |
| Motivation Quiz + Matcher | 4 days | ✅ **MUST HAVE** | Adds personalization; low complexity |
| Feedback Form + Analytics Logging | 2 days | ⚙️ **SHOULD HAVE** | Enables learning loop; simple to add |
| Export to LinkedIn | 1 day | 💡 **COULD HAVE** | Nice UX; not essential for demo |
| JD Truth Lens | 6 + days | 🚫 **WON’T HAVE** | Complex NLP; out of scope |

**Total Effort:** ≈ 14 days  
**Buffer:** + 2 days for testing & integration  

✅ Fits a 3-person × 3-week team.
